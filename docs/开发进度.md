### 📌 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 依赖管理 | PEP 621 `pyproject.toml`（标准格式） | 兼容 pip / uv / poetry，不绑定工具链 |
| Python 版本 | 3.11 | 兼顾性能与库支持，避免 3.13 的生态滞后 |
| ORM 风格 | SQLAlchemy 2.0 `DeclarativeBase` + `Mapped[]` | 类型友好，未来迁移到 async 低成本 |
| 数据库驱动 | `psycopg[binary]` (psycopg3) | 性能优于 psycopg2，原生 async 支持 |
| 日志库 | loguru | 配置极简，开发体验好 |
| LLM SDK | litellm 统一 | 一行切换 OpenAI / Anthropic / DeepSeek / Ollama |
| DB 镜像 | `pgvector/pgvector:pg16` | 免去 pgvector 自行编译，Step 2 无缝启用向量 |
| Lint | Ruff（一把梭代替 flake8/black/isort） | 快且配置简单 |
| 任务队列 | Celery + Redis | 成熟方案；Step 4 有需要可换 Arq / Dramatiq |

### ⚠️ 已知坑与注意事项

- **Windows 用户**：开发务必走 WSL2 + Docker Desktop，Celery + 中文路径在原生 Windows 常有诡异问题。
- **pgvector 镜像**：`pgvector/pgvector:pg16` 体积约 300MB，首次 pull 耐心等待。
- **`.env` 切勿提交**：已在 `.gitignore` 中，但仍要 `git status` 二次确认。
- **flushb 测试**：`test_health` 当 DB 不可用时 `db` 字段会返回 `error: xxx` 但整体仍 200，这是设计，避免 CI 因 DB 未就绪失败。
- **LLM 账单**：Step 0 完全不会产生任何 LLM 消耗；Step 1 起会开始调用，请先确认预算。

### 📌 Step 1 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 业务编排放在 `app/services/` | 与 collectors/analyzers/notifiers 解耦 | 让 Celery task 与 REST handler 共用同一份业务逻辑，避免重复 |
| LLM 调用统一走 `LLMClient.chat_json` | 单一入口 + tenacity 重试 + JSON 容错（多种正则） | 不同厂商返回格式偏差大，容错成本必须前置 |
| 卡片生成与 HTTP 发送分离 | `cards.py` 纯 dict，`feishu.py` 纯 HTTP | 卡片可直接单测，无需 mock HTTP |
| 推送幂等：`PushLog(channel, batch_key)` 唯一索引 | 日报 `daily:YYYY-MM-DD`、单文章 `article:{id}` | 重复触发不会洗版 |
| Alembic 配置放 `backend/alembic.ini` | `prepend_sys_path = .`，`script_location = migrations` | 容器 WORKDIR=/app/backend 时直接 `alembic upgrade head` 可用 |
| FastAPI 的 B008（`Depends` 默认参数） | 全局 ignore | 是 FastAPI 标准模式，无需绕路 |

### ⚠️ Step 1 已知坑

- **Celery beat 持久化**：`celerybeat-schedule` 文件在 backend 容器里生成，挂载到 host 也无害；如果改 cron 需要重启 beat 容器。
- **LLM JSON 输出**：实测某些模型会偶发返回 ```json\n{...}\n``` 包裹甚至前后带解释，已用 3 段正则 fallback；若仍偶发失败，tenacity 会自动重试 3 次。
- **trafilatura 抓不到正文**：会自动降级到 RSS 自带的 `summary` 字段，避免文章内容为空导致分析失败。
- **Ollama 模式**：`LLM_PROVIDER=ollama` 时 `LLMClient` 会把 `OPENAI_BASE_URL` 末尾的 `/v1` 自动剥离（litellm Ollama 适配器要求）。
- **种子源中文站**：`机器之心` / `量子位` 的 RSS 偶有不稳定，失败会写入 `Source.last_error`，不影响其他源。
